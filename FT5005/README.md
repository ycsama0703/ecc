# FT5005: Multimodal ECC and High-Frequency Returns

Working repository for Team 4's FT5005 publication-track project on multimodal earnings conference calls, audio, and high-frequency market reactions.

Current teacher-approved direction:
- Primary pilot: DJ30 firms with transcript text, audio, timed sentence segments, analyst data, and 5-minute stock data.
- Main target: post-call and within-call abnormal volatility around earnings conference calls.
- Core modalities: text, audio, and high-frequency returns.
- Optional extension: weakly supervised alignment for larger SP500 or SP1500 samples if only embeddings can be taken home.

Key project pivot:
- The earlier news-amplification draft is no longer the main path.
- The current plan follows the professor's March 10, 2026 multimodal ECC notes and the DJ30 data package shared by Kelvin and Yu Ta.
- Because the shared DJ30 package already includes timed sentence segments in `A4`, the safe first paper is now timestamp-aware multimodal ECC modeling, not purely latent alignment from scratch.
- The latest reply from Kelvin and Yu Ta also confirms that `A4` is noisy, `A2` only gives scheduled time, and audio features should be generated by us.

Current next-phase adjustment after the first report feedback:
- the next paper round must define the target, sample period, split rule, prior, feature groups, and benchmark ladder much more explicitly;
- the project should formally compare the current off-hours formulation with a cleaner during-ECC target family before freezing the headline `Y`;
- the next major contribution should be one small but real prior-aware method, not another broad feature or router expansion.

- the follow-up signature-taxonomy checkpoint then sharpens that interpretation further: every single shared-core row is not just “clarificatory,” but specifically `clarify+quant`, while the richer semantic tail decomposes into a cleaner operational/quantitative branch (`clarify+quant`, `quant`) and a noisier structural/strategic branch (`clarify+structural`, `structural`, `quant+structural`); this explains why the compact factor is high-precision but low-recall, and why the remaining `2025` semantic tail survives outside the tiny lexical core

Interpretation:
- The current structured panel is usable and already produces non-trivial validation signal.
- The negative test `R^2` indicates time drift or overfitting, which is expected at this stage and is exactly why the next step should move toward stronger temporal validation and richer text or audio features rather than adding more ad hoc structured variables.
- Naive bag-of-words text alone is weak on the current target.
- The first incremental gain comes from finance-aware event aggregation: `Q&A` structure, lexical uncertainty or guidance rates, A4 timing quality, and lightweight audio proxies.
- The next practical step is not a larger vocabulary model but better finance-style aggregation, role-aware dialogue features, and more robust temporal evaluation.
- The strict role-aware sequence baseline is worth keeping as the bridge to a later GRU or TCN model, but the current fuzzy mapping quality is not yet strong enough to make it the final main model.

Interpretation after the new hard baselines:
- the main empirical challenge is now clear: beat same-ticker historical volatility priors, not just beat a global mean or a sparse text baseline
- current text and audio additions still show unstable validation-to-test transfer relative to that hard identity benchmark
- this strengthens the case for identity-aware evaluation and for keeping the main story focused on honest incremental value rather than headline model complexity
- benchmark-inspired `Q&A` features are useful for validation-side ranking and scientific interpretability, but not yet for a clean test-side win
- prior-as-input is not a good enough modeling form on its own; residual-on-prior is better aligned with the forecasting question and should be the default incremental-value framing going forward
- the biggest adjustment from this round is conceptual rather than cosmetic:
  - raw post-call volatility level is dominated by firm-specific scale,
  - but off-hours incremental volatility shock is much more learnable from ECC features
- this gives the project a much stronger possible main paper framing:
  - predict volatility shock rather than raw level,
  - focus on non-regular-hours calls where information assimilation is cleaner
- the newest ablation result also forces a methodological adjustment:
  - once the target and subset are specified correctly, aggressive feature accumulation is not automatically helpful
  - the paper should therefore emphasise disciplined ablation and incremental value, not just larger feature unions
- the newest robustness run sharpens the integrity story further:
  - the result is stable across years and both off-hours regimes,
  - but its incremental gain is still highly concentrated in a few tickers,
  - so concentration reporting must remain explicit in any paper draft
- the newest gated prototype also clarifies the method gap:
  - a more complex prior-aware form is not automatically better,
  - and the project still lacks a stronger method contribution that reliably improves over the current residual ridge core
- the new unseen-ticker stress test improves the external-validity story:
  - the structured off-hours shock design still performs strongly when the test ticker is entirely unseen during training,
  - which reduces, but does not eliminate, the risk that the current result is only a within-firm identity effect
